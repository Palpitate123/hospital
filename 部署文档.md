# 医院预约挂号系统部署文档

## 一、环境要求

### 1.1 硬件要求
- CPU: 2核及以上
- 内存: 4GB及以上
- 硬盘: 20GB及以上

### 1.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 1.8+ | Java运行环境 |
| MySQL | 8.0+ | 数据库 |
| Redis | 5.0+ | 缓存服务 |
| Node.js | 14.x+ | 前端构建环境 |
| Maven | 3.6+ | Java项目构建工具 |
| npm | 6.x+ | Node包管理器 |

## 二、后端部署

### 2.1 数据库配置

#### 2.1.1 创建数据库
```sql
CREATE DATABASE hospital DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

#### 2.1.2 导入数据库脚本
```bash
# 导入表结构
mysql -uroot -p hospital < sql/hospital_schema.sql

# 导入测试数据
mysql -uroot -p hospital < sql/hospital_data.sql
```

### 2.2 修改配置文件

编辑 `hospital-backend/src/main/resources/application.yml`:

```yaml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hospital?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的数据库密码
    
  redis:
    host: localhost
    port: 6379
    password: 你的Redis密码(如果没有设置则留空)
    
jwt:
  secret: 你的JWT密钥(建议使用32位以上随机字符串)
  expiration: 86400000
  
file:
  upload:
    path: ./hospital-upload-files/
```

### 2.3 编译打包

```bash
cd hospital-backend
mvn clean package -DskipTests
```

打包完成后，会在 `target` 目录下生成 `hospital-backend-1.0.0.jar`

### 2.4 启动后端服务

#### 开发环境启动
```bash
cd hospital-backend
mvn spring-boot:run
```

#### 生产环境启动
```bash
java -jar hospital-backend-1.0.0.jar
```

#### 后台运行（Linux）
```bash
nohup java -jar hospital-backend-1.0.0.jar > hospital.log 2>&1 &
```

#### Windows服务启动
```bash
start javaw -jar hospital-backend-1.0.0.jar
```

## 三、前端部署

### 3.1 安装依赖

```bash
cd hospital-frontend
npm install
```

### 3.2 修改配置

编辑 `hospital-frontend/vue.config.js`:

```javascript
module.exports = defineConfig({
  transpileDependencies: true,
  lintOnSave: false,
  publicPath: '/',
  outputDir: 'dist',
  assetsDir: 'static',
  devServer: {
    port: 8081,
    open: true,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',  // 后端地址
        changeOrigin: true,
        pathRewrite: {
          '^/api': '/api'
        }
      }
    }
  }
})
```

### 3.3 开发环境运行

```bash
cd hospital-frontend
npm run serve
```

访问地址: http://localhost:8081

### 3.4 生产环境打包

```bash
cd hospital-frontend
npm run build
```

打包完成后，会在 `dist` 目录下生成静态文件。

### 3.5 Nginx部署（推荐）

#### 3.5.1 安装Nginx
```bash
# Ubuntu/Debian
sudo apt install nginx

# CentOS
sudo yum install nginx
```

#### 3.5.2 配置Nginx

创建配置文件 `/etc/nginx/sites-available/hospital`:

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 前端静态文件
    location / {
        root /var/www/hospital/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    # 后端API代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # 文件上传大小限制
    client_max_body_size 10M;
}
```

#### 3.5.3 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/hospital /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 六、常见问题

### 6.1 后端启动失败

**问题**: 端口8080被占用
```bash
# 查看占用端口的进程
netstat -ano | findstr :8080
# Windows终止进程
taskkill /F /PID <进程ID>
# Linux终止进程
kill -9 <进程ID>
```

**问题**: 数据库连接失败
- 检查MySQL服务是否启动
- 检查数据库用户名密码是否正确
- 检查数据库是否已创建

**问题**: Redis连接失败
- 检查Redis服务是否启动
- 检查Redis端口是否正确（默认6379）

### 6.2 前端启动失败

**问题**: npm install 失败
```bash
# 清除缓存后重新安装
npm cache clean --force
npm install
```

**问题**: 代理请求失败
- 检查后端服务是否启动
- 检查vue.config.js中的代理配置

### 6.3 登录失败

**问题**: 提示用户名或密码错误
- 确认数据库中已导入测试数据
- 确认密码哈希值正确（BCrypt格式）

## 七、生产环境优化建议

### 7.1 后端优化

1. **JVM参数优化**
```bash
java -Xms512m -Xmx1024m -jar hospital-backend-1.0.0.jar
```

2. **开启GZIP压缩**（application.yml）
```yaml
server:
  compression:
    enabled: true
    mime-types: application/json,text/html,text/xml,text/plain
```

3. **数据库连接池优化**
```yaml
spring:
  datasource:
    druid:
      initial-size: 10
      min-idle: 10
      max-active: 50
```

### 7.2 前端优化

1. **开启GZIP压缩**（Nginx）
```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript;
gzip_min_length 1024;
```

2. **静态资源缓存**
```nginx
location ~* \.(js|css|png|jpg|jpeg|gif|ico)$ {
    expires 7d;
    add_header Cache-Control "public, immutable";
}
```

### 7.3 安全建议

1. 修改默认管理员密码
2. 配置HTTPS（SSL证书）
3. 配置防火墙规则
4. 定期备份数据库

## 八、数据库备份

### 8.1 手动备份
```bash

```

